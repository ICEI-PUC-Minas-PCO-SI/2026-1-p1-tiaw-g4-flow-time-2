# Metodologia

## Controle de versão
O projeto utiliza **Git** como ferramenta de versionamento e o **GitHub** para hospedagem do repositório.  
As branches seguem a seguinte convenção:
- **main**: versão estável já testada
- **dev**: versão em desenvolvimento
- **testing**: versão em testes
- **unstable**: versão experimental

Os commits são realizados com mensagens claras e objetivas, e os merges passam por revisão da equipe.  
Tags são utilizadas para marcar versões importantes do software.  
A gestão de issues é feita com etiquetas padronizadas:
- **documentation**: melhorias ou acréscimos à documentação  
- **bug**: problemas em funcionalidades  
- **enhancement**: melhorias necessárias  
- **feature**: novas funcionalidades a serem implementadas

## Gestão de tarefas
A equipe utiliza o **GitHub Projects (Kanban)** para organizar e acompanhar o andamento das atividades.  
O quadro é dividido em:
- **Backlog**: tarefas planejadas  
- **Em andamento**: tarefas em execução  
- **Concluído**: tarefas finalizadas
